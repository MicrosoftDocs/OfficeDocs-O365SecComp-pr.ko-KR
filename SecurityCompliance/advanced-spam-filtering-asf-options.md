# <a name="advanced-spam-filtering--options"></a>고급 스팸 필터링 옵션

고급 스팸 필터링 옵션 메시지의 다양 한 콘텐츠 특성을 검사 하는 기능 관리자에 게 제공 합니다. 메시지에 이러한 특성의 현재 상태 (강화할 스팸으로 식별할 수 있도록 액) 메시지의 스팸 점수 증가 또는 스팸 메시지를 표시 합니다. ASF 옵션 스팸 메시지에서 일반적으로 발견 되는 HTML 태그 및 URL 리디렉션 등의 특정 메시지 속성을 대상으로 합니다.
  
ASF 옵션을 사용하도록 설정하는 것은 적극적인 스팸 필터링 방식이며 이러한 옵션을 통해 필터링되는 메시지는 가양성으로 보고할 수 없습니다. 정기적인 최종 사용자 스팸 알림을 통해 이러한 메시지를 확인하여 스팸 격리에서 복구할 수 있습니다. ASF 옵션마다 고유하며 ASF 옵션과 일치하는 메시지의 인터넷 헤더에 나타나는 X-헤더 텍스트를 통해 식별할 수 도 있습니다. 자세한 내용은 [스팸 방지 메시지 헤더](anti-spam-message-headers.md)를 참조하세요.
  
,을 설정 하거나 해제 하려면 ASF 옵션을 설정할 수 있습니다 콘텐츠 필터 정책을 편집 하는 경우에 모드를 테스트 합니다. 자세한 내용은 [스팸 필터 정책 구성](configure-your-spam-filter-policies.md)을 참조 하십시오. 테스트 모드는 **NDR 후방 분산**을 사용할 수 없으면 **SPF 레코드: 영구 실패**, **조건부 보낸사람 ID 필터링: 영구 실패**, 및 **대량 메일** 옵션입니다. 
  
> [!TIP]
>  환경에 따라 스팸 차단 기능을 최대한 활용하려면 테스트 모드에서 ASF 옵션을 사용해 보십시오. 특정 ASF 옵션에서 스팸이 검색되는 비율이 높은 고객의 경우에는 프로덕션 환경에서 이러한 옵션을 구현하기 전에 먼저 테스트를 하는 것이 좋습니다. >  피싱이 우려되는 조직에서는 **SPF 레코드: 영구 실패** 옵션을 설정하는 것이 좋습니다.

|**스팸으로 표시 섹션** <br/> |이 옵션을 사용하도록 설정하면 일치하는 메시지의 SCL(스팸 지수)이 9로 설정되어 스팸이 확실시됩니다. 메시지에 대해 수행되는 작업은 콘텐츠 필터 정책의 **높은 정확도 스팸** 설정과 일치합니다.<br/> ||

|HTML의 JavaScript 또는 VBScript  <br/> |이 설정을 사용하는 경우 HTML에서 JavaScript 또는 Visual Basic Script Edition을 사용하는 메시지가 스팸으로 표시됩니다. 이 두 스크립팅 언어는 HTML 메시지에서 특정 작업이 자동으로 수행되도록 하는 데 사용됩니다. 브라우저는 문서의 나머지 부분과 함께 스크립트를 구문 분석하고 처리합니다.  <br/> |X-CustomSpam: HTML의 Javascript 또는 VBscript 태그  <br/> |
|HTML의 Frame 또는 IFrame 태그  <br/> |이 설정을 사용 하는 경우, 포함 된 모든 메시지는 \<프레임\> 또는 \<IFrame\> HTML 태그를 스팸으로 표시 됩니다. 이러한 태그는 텍스트 또는 그래픽을 표시 하기 위한 페이지의 서식을 지정 하려면 HTML 메시지 또는 웹사이트에 사용 됩니다.  <br/> |X-CustomSpam: HTML의 IFRAME 또는 FRAME  <br/> |
