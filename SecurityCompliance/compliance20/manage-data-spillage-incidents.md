# <a name="managing-a-data-spillage-incident-in-microsoft-365"></a>Microsoft 365에서 데이터 액체 엎질렀는지 여부 문제를 관리합니다. 

신뢰할 수 없는 환경에 기밀 문서를 놓을 때 데이터 액체 엎질렀는지 여부 하 게 합니다. 데이터 액체 엎질렀는지 여부 문제를 검색 하는 경우에, 크기 및 위치는 액체 엎질렀는지 여부의 신속 하 게 평가 하 고,와 관련 된 사용자 활동을 검사 하 고, 다음 시스템에서 spilled 데이터를 영구적으로 제거 하는 것이 중요 합니다.

## <a name="scope-of-this-article"></a>이 문서의 범위

이 문서에서는 중이거나 응답할 더이상 액세스할 수 있는 사용자 또는 관리자가 복구할 수 있도록 Office 365 사서함에서 항목을 영구적으로 제거 하는 방법에 지침의 목록을 제공 합니다. 

> [!NOTE]
> 비즈니스 사이트에 대 한 SharePoint 또는 OneDrive에 있는 항목을 삭제 하면 원래 위치에서 삭제할 시간부터 93 일 동안 유지 합니다.

## <a name="scenario"></a>시나리오

여기서 직원 모른 매우 기밀 문서와 공유 전자 메일을 통해 여러 사용자 데이터 액체 엎질렀는지 여부 문제를 확인할는 있습니다. 내부 및 외부 조직에서이 문서를 받은 사람을 신속 하 게 평가 하려고 합니다. 인시던트를 조사 했을 때 후 그 결과 검토 하 고 다음 Office 365에서에서 spilled 데이터를 영구적으로 제거 하려면 다른 수 사관와 공유를 계획 합니다. 조사가 완료 되 면 모든 증거를 제거 하려고 합니다. 

## <a name="workflow"></a>워크플로

데이터 조사 (미리 보기)를 사용 하 여 데이터 액체 엎질렀는지 여부 문제를 관리 하는 것에 대 한 워크플로 다음과 같습니다.

1.  데이터 조사를 만듭니다.

2.  Spilled 데이터를 검색 합니다.

3.  검토 하 고 문제를 조사 합니다.

4.  Spilled 데이터를 영구적으로 삭제 합니다.

5.  닫거나 조사를 삭제 합니다.


## <a name="before-you-begin"></a>시작하기 전에

- Office 365 보안 & 준수 센터의에서 데이터 조사 (미리 보기) 도구를 사용 하 여 조사 만들기, spilled 데이터를 검색 하 고 검토 하 고 분석 합니다. 그런 다음 Office 365에서 spilled 데이터를 영구적으로 삭제 하는 보안 & 준수 센터 PowerShell을 사용 합니다. 

- 조사를 만들려면 보안 & 준수 센터의에서 규정 준수 관리자 역할 그룹의 구성원 이어야 해야 합니다.

- 메시지를 삭제 하려면 보안 & 준수 센터에서에서 조직 관리 역할 그룹의 구성원 이거나 검색 및 삭제 역할을 할당 해야 합니다. 역할 그룹에 사용자를 추가 하는 방법에 대 한 정보를 [보안 & 준수 센터에 대 한 사용자 액세스를 제공](../grant-access-to-the-security-and-compliance-center.md)을 참조 하십시오. 

- 어떤 사용자 사서함 및 OneDrive 계정을 조사 하는 사람을 검색할 수를 제어 하려면 조직 준수 경계와 설정할 수 있습니다. 자세한 내용은 [eDiscovery 조사에 대 한 준수 경계를 설정](../set-up-compliance-boundaries.md)합니다. 

## <a name="step-1-create-a-data-investigation"></a>1 단계: 데이터 조사 만들기

데이터 조사를 만들려면:

1. [https://protection.office.com](https://protection.office.com)으로 이동합니다.
    
2. 회사 또는 학교 계정을 사용하여 Office 365에 로그인합니다.
    
3. 보안 & 준수 센터에서에서 **데이터 조사**를 클릭 합니다.
 
4. **데이터 조사 (미리 보기)** 페이지에서 **새 조사 만들기**를 클릭 합니다.
    
5. **새 데이터 조사** 플라이 아웃 페이지 (필수) 이름을 조사를 지정 하 고는 선택적 조사 번호와 설명을 입력 합니다. 참고 이름은 조직 내에서 고유 해야 합니다.

6. 아래에서 **이 조사를 만든 후 추가 설정을 구성 하 시겠습니까?**, 다음 중 하나를 수행 합니다.

    - **예** 를 조사, 만들기를 클릭 하 고 새 경우에서 **설정** 페이지를 표시 합니다. 이 옵션을 사용 하면 조사에 구성원을 추가할 수 있습니다.
    
    - 방금 조사를 만들고 **데이터 조사 (미리 보기)** 페이지의 경우 목록에서 표시 하려면 **아니요** 클릭 합니다. 이 옵션을 선택 하면 추가 될 경우 조사 하 고 기본 검색 및 분석 설정의 유일한 구성원 사용 됩니다. 구성원을 추가 또는 조사를 만든 후 언제 든 지 설정을 변경할 수 있습니다.

7. 조사를 만들려면 **저장** 을 클릭 합니다.

    새 조사 **데이터 조사 (미리 보기)** 페이지의 목록에 표시 됩니다. 

8. 조사를 열려면 조사의 이름을 클릭 합니다. 

    **홈** 탭의 조사를 위해 표시 됩니다. 

> [!TIP]
> 조사에 대 한 명명 규칙을 설정 하는 것이 좋습니다. 하 고 찾을 하 고 필요한 경우를 나중에 참조할 수 있도록 이름 및 설명 수 만큼 정보를 제공 합니다.
 
## <a name="step-2-search-for-the-spilled-data"></a>Spilled 데이터에 대 한 2 단계: 검색 
 
넘어가 데이터에 대 한 검색 하려는 사용자를 알고 있는 경우에 조사 해당 데이터 원본에 매핑한 빠르게 검색 하 여 자신의 사서함 및 OneDrive 계정에 관심 있는 사람으로 추가할 수 있습니다. 경우 조사 관심 있는 사람을 추가 하려면 **관심 있는 사람**클릭 하 고 **관심 사람에 추가**클릭 합니다. 

**검색** 탭을 찾으려면 spilled 데이터 검색을 만들 수 있습니다. [4 단계에서](##step-4:-permanently-delete-the-spilled-data)에서 이러한 같은 메시지를 삭제 하려면 spilled 데이터를 찾는 데 사용 하는 동일한 검색 쿼리를 사용 합니다. 만들기 검색 하는 방법에 대 한 자세한 내용은 [만들기 데이터 수집에 대 한 검색을](create-search-to-collect-data.md)참조 하십시오.

검색을 실행 하면 검색 결과 및 보기 검색 통계 검색 쿼리와의 효율성을 평가 하는 예제를 미리볼 수 있습니다. Office 365에서 삭제 하려는 항목을 식별 한 후 수 **사고** 탭을 클릭 하 고 인시던트를 만들고 해당 항목을 포함 하는 검색 결과 추가 합니다. 

이 작업을 수행 하 게 조사 하려는 검색을 클릭 합니다. 플라이 아웃 페이지에서 **인시던트를 추가 결과** 클릭 하 고 지시를 따릅니다. 다음 사고, 개별 문서를 검토, 문서에 대 한 액세스를 있었을 조사 하 고 수는 문서를 내보냅니다. 단순히 검토 하는 대신 문서를 삭제 하려면 4 [단계로](##step-4:-permanently-delete-the-spilled-data)이동 합니다. 

> [!IMPORTANT]
> 검색 쿼리에 사용 하 여 키워드에 대 한 검색 하는 실제 spilled 데이터를 포함할 수 있습니다. 예를 사회보장 번호와 사용자를 포함 하는 문서에 대 한 검색 검색 쿼리에 키워드도이 옵션을 사용할 경우에 액체 엎질렀는지 여부를 방지 하려면 나중에 쿼리를 삭제 해야 합니다. 검색을 삭제 하거나 [5 단계에서](##step-5:-close-or-delete-investigation)에서 전체 조사를 삭제할 수 있습니다. 

## <a name="step-3-review-and-investigate"></a>3 단계: 검토 하 고 조사 

조사에 **사고** 탭으로 이동 하 고 이전 단계에서 만든 인시던트를 클릭 합니다. 처리 작업이 완료 되 고 검색 결과 사고에 추가 된, 후에 원시 형식으로, 텍스트 형식 또는 네이티브 근처 형식에서 개별 문서를 검토할 수 있습니다. 문서 및 사용자 조사에 대 한 결과 나타내기 위해 태그 문서 목록 범위를 더 좁힐 추가 쿼리를 만들 수 있습니다.

문서를 그룹화 하 고 검토를 위해 더 많은 도움을 받을 하려면 **관리 인시던트**를 클릭 합니다. **분석** 타일에서 **분석**을 클릭 합니다. 중복 검색, 스레딩, 전자 메일 및 테마 분석와 같은 고급 분석을 실행 됩니다. 자세한 내용은 다음을 참조 합니다.

- [중복 검색 근처](near-duplicates.md)
- [전자 메일 스레딩](email-threading.md)
- [테마](themes.md)

데이터 액체 엎질렀는지 여부에 관련 된 사용자를 확인 하려면 사고에서 새 쿼리를 만들 수 있으며 보낸사람/작성자 및 받는 사람 조건을 사용 하 여 다음 키를 누릅니다. 이렇게 하면 모든 보낸사람, 받는 사람 및 사고에 추가 된 수집 된 데이터에 있는 작성자 목록이 만들어집니다. 목록에 외부 사용자는 모두 있는지 확인 하려면 목록을 검토 해야 합니다. 자세한 내용은 [검색 조건](../keyword-queries-and-search-conditions.md#search-conditions)을 참조 하십시오.

## <a name="step-4-permanently-delete-the-spilled-data"></a>4 단계: spilled 데이터를 영구적으로 삭제

### <a name="deleting-mailbox-items"></a>사서함 항목을 삭제합니다.

검토 하 고 검색 결과 삭제 해야 하는 전자 메일 메시지를 포함 하는 유효성 검사 후 있습니다 영구적으로 삭제할 수를 실행 하 여는 **새로 ComplianceSearchAction--PurgeType HardDelete 항목 지우기** 보안 & 준수에에서 명령 센터 PowerShell 합니다. 자세한 내용은 [대 한 검색 및 삭제 전자 메일 메시지를](../search-for-and-delete-messages-in-your-organization.md)참조 하십시오. 

사용자의 복구 가능한 항목 폴더에 보관 함 (및 관리자가 액세스할 수 있는) 하면 조직에서 사서함에 대 한 단일 항목 복구를 사용 하는 경우 항목을 영구적으로 삭제 하는 내용의 됩니다 (기본값은 14 일) 삭제 된 항목 보존 기간 끝까지 합니다. 또한 넘어가 데이터를 포함 하는 사서함의 법적 보존 또는 보존 정책에 할당 된, 하는 경우 제거 된 메시지에에서 유지 됩니다 복구 가능한 항목 폴더 보존이 제거 시키거나 사서함 보존 정책에서 제외 됩니다. 하드 삭제 메시지에 즉시 추가 작업을 수행 해야 합니다. 자세한 내용은 [폴더에 있는 클라우드 기반 사서함의 대기 복구 가능한 항목에서 항목을 삭제 ](../delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md)합니다.  

> [!IMPORTANT]
> 보류 또는 보존 정책을 제거 하기 전에 레코드 관리 또는 법률 부서를 확인 합니다. 조직에서 사서함에 보존 여부를 정의 하는 정책을 이미 또는 데이터 액체 엎질렀는지 여부 문제가 발생 한 설정이 우선 합니다. 

### <a name="deleting-site-items"></a>사이트 항목 삭제

비즈니스 계정에 대 한 OneDrive 또는 SharePoint 사이트에서 문서를 영구적으로 삭제를 삭제 해야 하 고 사이트에서 삭제 한 다음 사이트 모음 휴지통에서에서 삭제 해야 합니다. 자세한 내용은 [SharePoint와 OneDrive에서 삭제 문서](https://docs.microsoft.com/microsoft-365/compliance/gdpr-dsr-office365#deleting-documents-in-sharepoint-online-and-onedrive-for-business)를 참조 하십시오.

또는 넘어가 데이터를 포함 수 있는 전체 사이트 모음을 삭제할 수 있습니다. 자세한 내용은 [사이트 모음을 삭제](https://docs.microsoft.com/sharepoint/delete-site-collection)합니다.

## <a name="step-5-close-or-delete-the-investigation"></a>5 단계:을 닫거나 조사를 삭제 합니다.

원본 콘텐츠 위치 (사서함 또는 사이트)에서 문서를 삭제 한 후에 조사 보안 문제에 추가 된 이러한 문서의 복사본 여전히 해야 합니다. 추가 데이터 액체 엎질렀는지 여부를 방지 하려면 조사 삭제 고려해 야 합니다.

조사를 삭제 하는 방법:

1. **설정** 탭에서 **조사 정보**를 클릭 합니다.
2. **대/소문자를 삭제**를 클릭 합니다. 

조사를 삭제할 필요가 없습니다 조사 중에 수집 된 정보를 저장 하려는 경우 **사례 닫기**을 클릭 합니다. 나중에 닫힌된 조사를 다시 열 수 있습니다.
