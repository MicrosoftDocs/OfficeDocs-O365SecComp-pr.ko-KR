## <a name="search-tips-and-tricks"></a><span data-ttu-id="8f5d6-680">검색 팁과 트릭</span><span class="sxs-lookup"><span data-stu-id="8f5d6-680">Search tips and tricks</span></span>

- <span data-ttu-id="8f5d6-p205">키워드 검색은 대/소문자를 구분하지 않습니다. 예를 들어 **cat** 및 **CAT**은 같은 결과를 반환합니다. </span><span class="sxs-lookup"><span data-stu-id="8f5d6-p205">Keyword searches are not case sensitive. For example, **cat** and **CAT** return the same results.</span></span> 
    
- <span data-ttu-id="8f5d6-683">부울 연산자 **AND**, **OR**, **NOT**, **NEAR**및 **onear** 는 대문자 여야 합니다.</span><span class="sxs-lookup"><span data-stu-id="8f5d6-683">The Boolean operators **AND**, **OR**, **NOT**, **NEAR**, and **ONEAR** must be uppercase.</span></span> 
    
- <span data-ttu-id="8f5d6-p206">두 개의 키워드나 두 `property:value` 식 사이의 공백은 **AND**를 사용 하는 것과 같습니다. 예를 `from:"Sara Davis" subject:reorganization` 들어 Sara Davis에서 보낸 모든 메시지를 제목 줄에 다시 그룹화 하 여 반환 합니다.</span><span class="sxs-lookup"><span data-stu-id="8f5d6-p206">A space between two keywords or two  `property:value` expressions is the same as using **AND**. - <span data-ttu-id="8f5d6-p207">`property:value` 형식과 일치 하는 구문을 사용 합니다. 값은 대/소문자를 구분 하지 않으며 연산자 뒤에 공백을 사용할 수 없습니다. 공백이 있는 경우 해당 값은 전체 텍스트 검색 일 뿐입니다. 예를 `to: pilarp` 들어 pilarp로 전송 된 메시지 대신 키워드로 "pilarp"을 검색 합니다.</span><span class="sxs-lookup"><span data-stu-id="8f5d6-p207">Use syntax that matches the  `property:value` format. For example `to: pilarp` searches for "pilarp" as a keyword, rather than for messages that were sent to pilarp.</span></span>
